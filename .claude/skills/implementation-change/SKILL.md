# Implementation Change Skill

Use for ordinary bug fixes or small features with clear validation.

Instructions:

1. Read `.agents/loops/implementation-change.md`.
2. Inspect relevant files and nearby tests before editing.
3. Make one minimal change per pass.
4. Run the narrowest meaningful check and use failures to choose the next pass.
5. Final answer must include changed files, validation results, and unrun checks.
